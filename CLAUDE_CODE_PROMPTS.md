## ЭТАП 5 — Авторизация /auth (вход по телефону)

```
Прочитай прикреплённые файлы CLAUDE_CODE_GUIDE.md + HTML файлы утверждённых страниц (blizhe_cards.html, free_landing.html, buy_landing.html).

Создай систему входа по номеру телефона без пароля (magic link / SMS код).

Алгоритм:
1. /auth — страница с полем ввода телефона
2. Пользователь вводит номер → POST /auth/send
3. Сервер генерирует 4-значный код, сохраняет в БД (SQLite) с TTL 10 минут
4. SMS с кодом отправляется через [SMS провайдер — уточни какой использовать]
5. Редирект на /auth/verify?phone=НОМЕР
6. /auth/verify — страница с полем ввода кода
7. Пользователь вводит код → POST /auth/verify
8. Если код верный — создать сессию (cookie на 30 дней), редирект на /cards
9. Если код неверный — показать ошибку

БД (SQLite):
- Таблица users: id, phone, created_at, paid_at
- Таблица sms_codes: id, phone, code, expires_at, used
- Таблица sessions: id, user_id, token, expires_at

Стиль страниц: в духе /free и /buy — те же цвета, шрифты, кнопки border-radius 50px.
Поля ввода: background --white, border 1px solid --beige, border-radius 12px.
```

## ЭТАП 6 — Webhook ЮКассы + Google Sheets

```
Прочитай прикреплённые файлы CLAUDE_CODE_GUIDE.md + HTML файлы утверждённых страниц (blizhe_cards.html, free_landing.html, buy_landing.html).

Создай обработчик webhook от ЮКассы и запись в Google Sheets.

POST /webhook/payment:
1. Принять JSON от ЮКассы
2. Проверить подпись (HMAC)
3. Если статус «succeeded»:
   a. Достать email и utm_source из metadata платежа
   b. Найти или создать пользователя в БД по email
   c. Записать paid_at = now()
   d. Записать строку в Google Sheets:
      Дата | Email | UTM | Сумма | Комиссия 30% (=сумма*0.3)
4. Вернуть 200 OK

Google Sheets:
- Использовать Google Sheets API v4
- Service Account для авторизации
- ID таблицы передаётся через переменную окружения GOOGLE_SHEET_ID
- Лист: «Продажи»
- Колонки: Дата | Email | UTM | Блогер | Сумма | Комиссия 30%
- Блогер = UTM (или «direct» если UTM пустой)

Переменные окружения (добавить в .env):
YUKASSA_SECRET_KEY=...
GOOGLE_SHEET_ID=...
GOOGLE_CREDENTIALS_JSON=...  (путь к файлу service account)
```

## ЭТАП 7 — Админка /admin

```
Прочитай прикреплённые файлы CLAUDE_CODE_GUIDE.md + HTML файлы утверждённых страниц (blizhe_cards.html, free_landing.html, buy_landing.html).

Создай простую админку для редактирования карточек без кода.

/admin — защищена паролем из переменной окружения ADMIN_PASSWORD.

Интерфейс:
- Три вкладки: Действие / Вопрос / Забота
- Каждая карточка — аккордеон (закрыт по умолчанию)
- В аккордеоне — поля textarea для каждого поля карточки:
  * Уровень (select: Лёгкий / Средний / Глубокий)
  * Текст карточки (textarea, высота ~120px)
  * Подсказка (textarea, высота ~60px)
  * Зачем это (textarea, высота ~100px)
  * Мозг (textarea, высота ~100px) — каждый пункт с новой строки
  * Наука (textarea, высота ~100px)
  * К чему приведёт (textarea, высота ~80px)
- Кнопка «Сохранить» под каждой карточкой → PATCH /admin/card/<id>
- Изменения сохраняются в JSON файл и сразу видны пользователям

ВАЖНО: При сохранении никогда не обрезать и не изменять текст — только записывать то, что в поле.

Стиль: минималистичный, функциональный. Те же цвета и шрифты что на основном сайте.
Кнопки border-radius 50px. Поля ввода: background --white, border --beige, border-radius 12px.
```
